# 贡献指南

感谢您对知更鸟战舰（Robin Fleet）项目的关注！我们欢迎各种形式的贡献，包括但不限于：报告Bug、提交功能建议、改进文档、完善Demo等。

## 贡献方式

### 1. 报告问题（Issues）

- 使用 GitHub Issues 提交 Bug 报告或功能请求。
- 请先搜索已有 Issues，避免重复提交。
- 提交时请附上以下信息：
  - 问题描述（清晰、简洁）
  - 复现步骤（如适用）
  - 期望行为与实际行为
  - 环境信息（浏览器版本、操作系统等）

### 2. 提交代码（Pull Requests）

- Fork 本仓库，创建您的特性分支（`git checkout -b feature/your-feature`）。
- 确保代码风格与现有代码一致，适当添加注释。
- 提交前确保没有破坏现有功能（运行 `index.html` 验证）。
- 提交 Pull Request 时，请详细描述您的改动原因和影响。

### 3. 改进文档

- 文档存放在 `docs/` 目录下，欢迎修正错别字、补充说明、完善示例。
- 对于 README 等核心文档，请保持语言简洁、结构清晰。

### 4. 参与比赛材料

- 如果您希望协助完善比赛材料（如 `goai-2026/` 目录下的文件），请通过 Issues 沟通。

## 开发环境

- 本项目为纯前端静态展示仓库，无需后端环境。
- 可直接在浏览器中打开 `index.html` 运行，或使用本地 HTTP 服务器（如 `python -m http.server 8000`）。

## 代码规范

- HTML 文件：遵循 HTML5 标准，语义化标签。
- CSS 文件：使用 BEM 命名规范（如适用）。
- JavaScript 文件：ES6+ 语法，避免全局变量。

## 提交信息规范

- 使用英文或中文描述，短句形式。
- 前缀参考：
  - `feat:` 新功能
  - `fix:` Bug 修复
  - `docs:` 文档更新
  - `style:` 代码格式调整
  - `refactor:` 重构
  - `test:` 测试相关
  - `chore:` 构建、CI 等

## 行为准则

- 尊重所有参与者，文明交流。
- 禁止任何形式的歧视、骚扰或人身攻击。
- 关注技术本身，保持友善的讨论氛围。

## 联系我们

- 如有合作或授权需求，请通过 GitHub Issues 留言。
